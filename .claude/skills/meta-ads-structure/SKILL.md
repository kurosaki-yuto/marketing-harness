# Meta Ads 構造

## 3 階層の役割

```
キャンペーン
├── 目標（認知/トラフィック/リード/売上 等）を設定
├── 予算戦略（CBO or ABO）を設定
└── 広告セット
    ├── ターゲット（オーディエンス）
    ├── 配信面
    ├── スケジュール・入札上限
    └── 広告
        ├── クリエイティブ（画像/動画/カルーセル）
        ├── テキスト・見出し・CTA
        └── 遷移先 URL
```

## CBO vs ABO の使い分け

| 項目 | CBO | ABO |
|---|---|---|
| 予算配分 | AI が広告セット間を自動配分 | 広告セットごとに手動設定 |
| 向き | スケール済み・勝ちパターンあり | テスト段階・均等配分したい |
| メリット | 効率的。勝ちターゲットに集中 | 各セットのデータが均等に取れる |
| デメリット | 弱いセットが予算を受け取れない | 非効率になりがち |

**推奨フロー**: ABO でテスト → 勝ちセット確認 → CBO でスケール

## オーディエンスの種類と優先度

1. **カスタムオーディエンス（最優先）**: サイト訪問者・購入者リスト・エンゲージメントユーザー
2. **類似オーディエンス（LAL）**: 既存顧客の 1〜3% LAL が最も効率良い
3. **詳細ターゲット**: 興味関心・行動・デモグラ。競合が多く CPA が高めになりがち

## 配信面の選択

- **アドバンテージ + 配信（自動）**: AI が最適な配信面を選ぶ。基本は ON 推奨
- **手動配置**: クリエイティブのアスペクト比を配信面ごとに最適化したいときに使う

主要配信面:
- フィード（1:1 or 4:5）: リーチ量が最大
- ストーリーズ・リール（9:16）: 若年層・高エンゲージメント
- Instagram 発見タブ: 新規リーチ

## 典型的な構成例（EC）

```
認知キャンペーン（リーチ目標）
└── 広告セット: 類似 1-3% + 詳細ターゲット（ABO）

コンバージョンキャンペーン（売上目標）
├── 広告セット: サイト訪問者 → 非購入（7 日以内）← リターゲ
└── 広告セット: カート追加 → 非購入（3 日以内）← リターゲ高優先
```

## 入札戦略の選択

- **最低コスト（自動）**: 予算内で最大コンバージョン。スケール時に使う
- **コスト上限**: CPA の上限を設定。学習完了後に安定した数字が出てから使う
- **ROAS 目標**: EC で ROAS を維持したいときに使う
