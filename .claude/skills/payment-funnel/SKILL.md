# 決済ファネル設計

## 決済フローの基本構成

```
商品ページ
→ カート追加
→ カート確認（商品・数量）
→ 会員登録 or ゲスト購入
→ 配送先・支払い情報入力
→ 注文確認
→ 決済完了
→ サンキューページ（アップセル機会）
```

各ステップの離脱率を計測し、最も離脱が多いステップを重点改善する。

## カート放棄対策

**EC 平均カート放棄率: 70〜75%**。放棄メール 3 部作が最も有効。

### 放棄メールシリーズ
1. **1 時間後**: 「カートに忘れ物ですよ」（リマインド）
2. **24 時間後**: 「まだ迷っていますか？」+ よくある疑問に回答
3. **72 時間後**: 「最後のご案内」+ 特別クーポン（在庫切れ懸念を添える）

放棄メールの開封率は通常メールの 2〜3 倍。CVR は 5〜15% が目標。

## アップセル・クロスセルの設計

### オーダーバンプ（決済ページ内）
決済フォームの横や下に表示する追加購入オファー。

- チェックボックス 1 つで追加購入できる摩擦の低さが強み
- 元商品の補完品（消耗品・保証・特典追加）が最も成約しやすい
- 追加単価は元商品の 10〜30% が適切

### アップセル（サンキューページ・決済後）
購入直後は購買意欲が最高潮。サンキューページに次のオファーを置く。

- 「ご購入ありがとうございます。特別に〇〇を今だけご案内します」
- **ワンクリック購入**（カード情報再入力不要）が成約率 2〜3 倍
- 元商品の上位版・関連商品を提示

### ダウンセル
アップセルを断った人に、より低価格な別のオファーを提示。

- 「〇〇は今回は不要でしょうか？では、〇〇はいかがですか？」
- 完全な離脱より、低価格でも成約した方が LTV が上がる

## 支払い方法の最適化

| 支払い方法 | CVR への影響 |
|---|---|
| クレジットカード | 標準 |
| コンビニ払い | CVR 低下気味だが高額商品に有効 |
| 後払い（BNPL） | CVR +10〜20%（不安が下がる） |
| 分割払い（月払い） | 高単価商品で CVR +30〜50% |
| 銀行振込 | リターン率が低い |

### 分割払いの設計
- 月額 ¥3,000 程度に落とすと心理的ハードルが激減
- 「¥120,000 一括」より「月々 ¥9,800 × 12 回」の方が成約しやすい
- 分割でも手数料なし（売り手負担）にするとさらに成約率 UP

## 決済フォームの最適化

- **入力項目を最小化**: 不要なフィールドを削除（電話番号は必要か？）
- **住所自動補完**: 郵便番号から自動入力
- **エラー表示**: リアルタイムバリデーション（送信後ではなく入力中に）
- **進行状況表示**: 「ステップ 2/3」で安心感を与える
- **セキュリティバッジ**: SSL・決済代行ブランドのロゴを表示

## KPI

| 指標 | 目標 |
|---|---|
| カート追加率 | 商品ページ PV の 5〜15% |
| カート → 購入完了率 | 30〜50% |
| アップセル成約率 | 5〜30% |
| 放棄メール CVR | 5〜15% |
