# KPI 設計

## 指標設計の考え方

KPI は「目標から逆算」する。

```
LTV（顧客生涯価値）
→ 許容 CPA = LTV × 粗利率 × 許容回収期間
→ 目標 CVR から目標 CPC を逆算
→ CPC から目標 CTR と CPM を逆算
```

## 業種別 KPI 目安

### EC
| 指標 | 目安 |
|---|---|
| ROAS | 200〜400%（粗利に依存） |
| CPA | 商品単価の 10〜30% |
| CVR | 1〜3% |
| CTR | 1.0〜2.5% |
| カゴ落ち率 | 70〜75%（業界平均） |

### BtoB（リード獲得）
| 指標 | 目安 |
|---|---|
| CPA（リード） | ¥3,000〜¥30,000 |
| リード → 商談率 | 10〜30% |
| 商談 → 受注率 | 20〜40% |
| CAC（顧客獲得コスト） | 月額単価 × 3〜12 ヶ月分 |

### サブスク（SaaS / 定期購入）
| 指標 | 目安 |
|---|---|
| CAC | MRR × LTV 月数 ÷ 粗利率 以内 |
| LTV/CAC | 3 以上が健全 |
| チャーンレート | 月 2% 以下を目指す |
| 初月 CPA | 月額の 3〜6 ヶ月分以内 |

### 店舗集客
| 指標 | 目安 |
|---|---|
| 来店 CPA | ¥500〜¥3,000 |
| CPM | ¥500〜¥1,200 |
| リーチ率（商圏内） | 月 30% 以上を目標 |

## KPI 閾値の設計パターン

### 3 段階閾値
```
目標値（例 CPA ¥2,000）
├── 警告: 目標 × 1.3 = ¥2,600 → アラート・分析
├── 危険: 目標 × 1.5 = ¥3,000 → 即改善アクション
└── 停止: 目標 × 2.0 = ¥4,000 → キャンペーン停止検討
```

### 日次 vs 週次 vs 月次
- **日次チェック**: 予算消化率・CPA 速報（スパイク検知）
- **週次チェック**: CPA トレンド・CTR 変化・クリエイティブ別パフォーマンス
- **月次チェック**: ROAS・LTV・ROI の総合評価

## marketing-harness での KPI 設定

MCP ツール `set_kpi_target` に渡すべきフィールド:
```json
{
  "campaign_id": "キャンペーンID",
  "targets": {
    "cpa": 2000,
    "roas": 300,
    "ctr": 1.5,
    "cpc": 100
  },
  "thresholds": {
    "cpa": 3000,
    "roas": 200
  }
}
```
thresholds を超えたとき ALERT_WEBHOOK_URL に通知が飛ぶ。
