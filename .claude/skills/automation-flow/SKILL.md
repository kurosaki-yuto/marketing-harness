# マーケティングオートメーション設計

## MA の基本概念

「誰に」「いつ」「何を」送るかを事前に設計して自動実行する仕組み。

**目標**: 人の手をかけずに、見込み客を「認知 → 関心 → 検討 → 購入 → ロイヤル」に育てる。

## ファネル別シナリオ設計

### リードナーチャリング（見込み客育成）

```
[流入] 広告 / SNS / 検索
  ↓
[登録] LP でメール登録（マグネット特典）
  ↓
[ウェルカム] 即日: ブランド紹介 + 特典配信
  ↓
[教育] 3〜7 日: 問題提起 → 解決策 → 事例の 3 部作
  ↓
[オファー] Day7〜14: 初回購入オファー（期間限定）
  ↓
[分岐]
  購入した → 既存顧客シナリオへ
  未購入  → フォローアップ + 別角度のオファーへ
```

### 既存顧客フォロー

```
[購入直後] 感謝 + 使い方ガイド
  ↓
[7 日後] 活用状況確認 + ヒント提供
  ↓
[30 日後] 満足度確認 + レビュー依頼
  ↓
[60 日後] リピート購入 or アップセルオファー
  ↓
[90 日後] 紹介プログラムの案内
```

## 行動トリガーの種類

| トリガー | 内容 | 活用例 |
|---|---|---|
| 時間経過 | 登録から N 日後 | ステップメール |
| ページ閲覧 | 特定 URL を訪問 | 商品ページ閲覧 → 関連メール |
| メール開封 | メールを開封した | エンゲージメント高い人に限定オファー |
| クリック | メール内リンクをクリック | 興味あり → 詳細案内 |
| 購入 | 商品を購入 | サンキュー + アップセル |
| 未開封 | N 日間開封なし | 件名を変えて再送 |
| カート放棄 | カート追加後 N 時間放置 | 放棄メール |

## セグメント設計

**RFM 分析**で顧客を分類してシナリオを分ける:
- **R（Recency）**: 最終購入からの日数
- **F（Frequency）**: 購入回数
- **M（Monetary）**: 購入金額合計

| セグメント | 定義 | アプローチ |
|---|---|---|
| VIP | R 低・F 高・M 高 | 先行案内・特別優遇 |
| 復活候補 | R 高・F 高・M 高 | 「お久しぶりです」特典 |
| 新規育成 | R 低・F 低 | ナーチャリングシナリオ |
| 休眠 | R 高・F 低・M 低 | ウィンバック or リスト削除 |

## シナリオ設計の原則

1. **1 メール 1 メッセージ**: 複数の訴求を 1 通に詰め込まない
2. **一貫したトーン**: シナリオを通して同じキャラクター・語り口を維持
3. **逃げ道を作る**: 各メールに「購読解除」を明記（法的要件＋信頼感）
4. **数字で検証**: 開封率・CVR を KPI として週次でモニタリング
5. **定期的な見直し**: 6 ヶ月ごとにシナリオを全体再設計する
