# LP コンバージョン最適化

## ファーストビューの 3 要素

画面に入ってきた瞬間（3 秒以内）に「自分ごと」と思わせる。

1. **ヘッドライン**: 誰のための何か・最大のベネフィットを一言で
   - NG: 「高品質な広告運用ツール」（機能説明）
   - OK: 「広告費を半分にしても成果が出るツール」（ベネフィット）

2. **サブヘッドライン**: ヘッドラインを補足。具体的な数字や条件
   - 例: 「月 5 万円以下の広告費でも ROI 300% を達成した 200 社の実績」

3. **ビジュアル**: 商品・使用シーン・人物。顔写真は信頼感を上げる

**ファーストビューに CTA ボタンを 1 つ置く**（スクロールを促す前に）

## LP の基本構成

```
[ファーストビュー] キャッチコピー + CTA
[問題提起] 読者の悩み・課題を言語化
[解決策] 商品・サービスの概要
[証拠・実績] お客様の声・数値・メディア掲載
[詳細・特徴] 機能・内容・差別化
[FAQ] よくある疑問・不安解消
[クロージング] 最後の CTA（期間限定・特典）
[フッター] プライバシーポリシー・会社情報
```

## CTA ボタンの設計

**コピー**:
- 「今すぐ購入」より「まず無料で試す」「詳細を確認する」の方がクリックされやすい
- ボタン内に価値を入れる: 「30 日間無料で始める」

**デザイン**:
- 背景と対比の強い色（オレンジ・緑が多い）
- 指で押しやすい大きさ（スマホで最低 44px）
- ホバー・タップ時に視覚的フィードバック

**配置**:
- ファーストビュー・各セクションの終わり・ページ下部に配置
- スクロール量の多い LP は「固定フッター CTA」が有効

## 離脱防止策

1. **表示速度**: 3 秒以内に表示されないと 53% が離脱（モバイル）。画像圧縮・CDN 必須
2. **モバイル最適化**: 70% 以上がスマホ閲覧。モバイルファーストで設計
3. **信頼シグナル**: ロゴ・受賞・メディア掲載・利用者数を Early on で見せる
4. **チャット・お問い合わせ**: 迷っている人への逃げ道を作る
5. **ポップアップ（離脱検知）**: 直帰しようとした人にクーポン・特典を提示

## A/B テストの優先順位（インパクト大きい順）

1. ヘッドライン
2. ファーストビューのビジュアル
3. CTA ボタンのコピー・色
4. 証拠セクションの構成
5. ページ全体のレイアウト

## 計測・改善サイクル

- **ヒートマップ**: どこまでスクロールされているか・どこをクリックしているか
- **フォームの途中離脱**: どのフィールドで離脱しているか
- **デバイス別 CVR**: PC と SP で乖離が大きい場合は SP 優先で改善

## 目標 CVR の目安

| 目的 | CVR 目安 |
|---|---|
| メール登録（無料） | 10〜30% |
| 無料トライアル | 5〜15% |
| 商品購入（EC） | 1〜5% |
| 問い合わせ・相談申込 | 2〜8% |
