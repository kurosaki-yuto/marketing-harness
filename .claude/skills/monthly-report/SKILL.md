# 月次広告レポート 作成ガイド

## レポート構成（標準テンプレート）

```
1. エグゼクティブサマリー（1 段落）
2. 月次 KPI サマリー（表）
3. キャンペーン別パフォーマンス
4. クリエイティブ分析
5. オーディエンス分析
6. 改善提案（来月のアクション）
7. 付録（生データ）
```

## 1. エグゼクティブサマリーの書き方

3 文以内で「結論 → 根拠 → 次のアクション」を書く。

例:
> 今月の CPA は ¥1,850（目標比 +8%）と目標内に収まりました。リターゲティング広告が CPA ¥1,200 と好調で、認知キャンペーンが全体を牽引しました。来月はリターゲティング予算を 30% 増額し、スケールを図ります。

## 3. 比較軸の選び方

- **前月比**: 季節性を含む短期トレンドを見る
- **前年同月比**: 季節性を除いた成長率を見る（最重要）
- **目標比**: 予算計画に対する達成状況
- **業界ベンチマーク比**: 絶対値の善し悪しを判断

## 4. キャンペーン別分析

各キャンペーンを「費用シェア × CPA」でプロットして評価:
- 費用多い × CPA 低い → 勝ちキャンペーン。予算増額を検討
- 費用多い × CPA 高い → 要改善または縮小
- 費用少ない × CPA 低い → スケールアップ候補
- 費用少ない × CPA 高い → 停止候補

## 5. 改善提案の書き方

SMART 形式で書く（Specific / Measurable / Achievable / Relevant / Time-bound）。

良い例:
> 来月は認知キャンペーンのクリエイティブを 3 パターン刷新し、CTR を現状の 0.8% から 1.2% 以上に改善します。

悪い例:
> 来月はクリエイティブを改善します。

## marketing-harness での生成フロー

1. `create_report` MCP ツールで月次データ取得
2. `list_knowledge` で過去の改善ナレッジ参照
3. 上記テンプレートに当てはめてレポート作成
4. 改善提案を `update_knowledge` でナレッジとして保存
