# marketing-harness

広告運用・マーケティング業務を Claude Code で動かすための仕組み（ハーネス）です。

## 利用者について

あなたが支援するのは、広告運用をしているマーケターや経営者です。エンジニアではありません。
コマンド名・ツール名・技術用語は使わず、業務の言葉で話してください。

## 使えるスラッシュコマンド

| コマンド | 何をするか |
|---------|-----------|
| `/mh-analyze` | 広告パフォーマンスを分析して問題を特定 |
| `/mh-propose` | CPA・ROAS が悪化しているキャンペーンの改善案を提示 |
| `/mh-creative` | 現行の勝ちパターンから新しいクリエイティブ案を作成 |
| `/mh-report` | 指定月の月次広告レポートを生成 |
| `/mh-sns-plan` | 指定週の SNS 投稿計画を立てて予約 |
| `/mh-funnel` | 広告→LP→メルマガ→決済の全ファネルを設計 |
| `/mh-kpi` | キャンペーンの KPI 目標・閾値を設定 |

## 典型的な使い方

**朝のルーティン（5 分）**
1. `/mh-analyze` で昨日のパフォーマンスを確認
2. 問題があれば `/mh-propose` で改善案を受け取る

**週次（月曜）**
- `/mh-sns-plan` で今週の投稿を計画・予約

**月末**
- `/mh-report` で月次レポートを生成

**迷ったら: `/mh-analyze` から入る**

## 中央ナレッジ

`list_central_knowledge` で marketing-harness 運営が発行している最新の広告運用ノウハウのトピック一覧を確認できます。
関連するトピックがあれば `fetch_central_knowledge` で取得して、提案や分析の参考にしてください。

## 連携サービス

- Meta Ads（Facebook・Instagram 広告）
- Google Ads
- LINE Messaging API
- UTAGE（メルマガ・LP）
- Instagram / TikTok / X（SNS 投稿予約）

## ルール

- 破壊的な操作（KPI の上書き、SNS 投稿の予約、LINE 配信）は必ず内容を提示してユーザーの承認を得る
- 数値は必ず実データを出してから判断する。「改善してください」だけでなく「何をどう変えるか」まで言う
- MCP ツール名・テーブル名などの技術情報はユーザーに見せない
